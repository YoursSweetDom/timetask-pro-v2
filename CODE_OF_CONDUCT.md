# Code of Conduct

TimeTask Pro V2 should be a respectful and practical open source space. People should be able to report issues, ask questions, review code, and disagree without being attacked or dismissed.

## Expected Behavior

- Be respectful and constructive.
- Assume good intent when possible.
- Give clear technical feedback.
- Keep discussions focused on the project.
- Credit ideas and contributions fairly.
- Welcome people with different experience levels.

## Unacceptable Behavior

- Harassment, threats, insults, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without permission.
- Repeated off-topic disruption.
- Bad-faith issue spam or pull requests.
- Pressuring maintainers to accept unsafe or misleading changes.

## Enforcement

Maintainers may remove comments, close issues, block users, or reject contributions that violate this code of conduct. Serious security or abuse concerns should be reported privately through the process in `SECURITY.md`.

## Scope

This code of conduct applies to project spaces such as GitHub issues, pull requests, discussions, documentation, and maintainer communication connected to TimeTask Pro V2.
